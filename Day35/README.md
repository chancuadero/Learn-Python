Topic: Stacks
Summary: Stacks follows the LIFO ordering principle, which stands for Last-In-First-Out. It means that the last inserted item will be the first item to be removed.

Topic: Queues
Summary: Queues are a data structure that follows the FIFO ordering principle, which stands for First-In-First-Out. The first inserted item is the first to be removed. The beginning of the queue is called the head, and the end is called the tail. In queues, as in stacks, we can only insert data at the end of the queue. This operation is called enqueue. However, contrary to stacks, we can only remove data from the head. This operation is called dequeue.
